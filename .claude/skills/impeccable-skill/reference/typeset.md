# Typeset：改善字体排版

> "好的字体排版是无形的；坏的字体排版是分心的。" 用有意图的、与品牌对齐的字体替换通用默认值，使用有意识的对比和可扩展的尺寸。

---

## 注册类型

**Brand（品牌型）**：用反映品牌的字体替换通用默认值（Inter、Roboto、平面比例的系统回退）。按 `brand.md` 运行字体选择。相邻级别比例 ≥1.25 的流体 `clamp()` 比例。

**Product（产品型）**：系统字体和熟悉的 sans 栈在这里工作良好。一个调优的家族用固定 rem 比例和 1.125–1.2 相邻级别比例承载 UI。

---

## 评估当前字体排版

**字体选择**：
- 当个性重要时，是否使用了无形的默认值（Inter、Roboto、Arial、Open Sans、系统默认值）？
- 字体是否匹配品牌个性？
- 字体家族太多了？（超过 2–3 个会产生问题）

**层级**：
- 标题、正文和说明是否一目了然就能区分？
- 字体大小是否太相似？（14px、15px、16px = 模糊的层级）
- 字重对比是否足够强？

**尺寸与比例**：
- 有一致的字体比例还是任意尺寸？
- 正文文本是否满足最小可读性（16px+）？
- 尺寸策略是否合适？（应用 UI 固定 rem；营销标题流体 `clamp()`）

**可读性**：
- 行长舒适（45–75ch 理想）？
- 行高是否适合字体和上下文？
- 文本与背景对比是否足够？

**一致性**：
- 相同角色的元素在整个项目中样式相同吗？
- 字重是否按角色一致使用？
- 字间距是有意的还是默认的？

## 系统性地改善字体排版

### 字体选择

- 选择反映品牌个性的字体
- 用真正对比度配对（衬线 + 无衬线、几何 + 人文主义）或单一家族多字重
- 确保 Web 字体防止布局移动：`font-display: swap`、度量匹配的降级

### 建立层级

五个尺寸覆盖大多数需求：说明、次要、正文、副标题、标题。

在级别之间使用一致的比例（1.25、1.333 或 1.5）。"结合维度：尺寸 + 字重 + 颜色 + 空间以建立强层级。不要仅依赖尺寸。"

- 应用 UI：固定 rem 比例， optionally 在 1–2 个断点调整
- 营销/内容：标题用流体 `clamp()`；正文文本固定

### 修复可读性

- 文本容器上 `max-width: 65ch`
- 行高：标题更紧（1.1–1.2），正文更松（1.5–1.7）
- 浅色在深色文本上略微增加行高
- 正文文本最小 16px / 1rem
