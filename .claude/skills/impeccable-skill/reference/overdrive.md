# Overdrive：超越常规极限

> "上下文决定'非凡'意味着什么。粒子系统适合创意作品集，但不适合设置页面——尽管带有即时乐观保存和动画状态转换的设置面板本身就很非凡。"

**这个命令有最高的误操作风险。** 在构建之前提出建议。

## 在构建之前提出建议

思考 2–3 个方向，具有不同的技术和审美方法。解释权衡。在写任何代码之前获得用户确认。

---

## 评估"非凡"在这里意味着什么

**视觉/营销表面**：感官 wow——动画序列、视差、维度效果、氛围纹理。

**功能性 UI**：它*感觉*如何——即时反馈、平滑状态转换、物理-like 交互、令人满意的微时刻。

**性能关键 UI**：无形但可感觉——感觉原生的骨架状态、乐观更新、零卡顿交互。

**数据密集界面**：流动性和响应性——实时更新图表、平滑筛选、感觉无限的虚拟化滚动。

**共同线索**：每种技术都必须让用户体验真正更好，而不仅仅是技术上令人印象深刻。

---

## 工具包

**电影过渡**：View Transitions API 用于状态之间的元素变形；跨页面的共享元素过渡。

**滚动绑定动画**：`IntersectionObserver` 用于触发点；通过 CSS `animation-timeline: scroll()` 的滚动驱动动画。

**超越 CSS**：WebGL/Canvas 用于 CSS 无法表达的效果；GSAP 用于复杂协调序列。

**数据感觉活着**：数据更新时动画的基于 Canvas 的图表；状态之间的平滑插值。

**复杂属性动画**：Clip-path 变形、SVG 路径动画、过滤器过渡（模糊、亮度、色相旋转）。

**性能优势**：10,000+ 项的虚拟滚动；用于后台渲染的 `OffscreenCanvas`；用于重计算的 Web Workers。

**设备集成**：`DeviceMotionEvent` 用于倾斜/陀螺仪；`matchMedia` 用于深色模式；`getBattery()` 用于减少效果模式。

## 有纪律地实现

**渐进增强是不可妥协的**：每种技术必须优雅降级。首先在没有增强的情况下测试——如果基础体验坏了，先修复那个。

**性能规则**：
- 目标 60fps；如果低于 50fps 就简化
- 在真实的中档设备上测试（不仅仅是你的 M 系列 Mac）
- 将昂贵效果绑定到小区域（模糊、过滤器、阴影）
- `will-change` 仅在动画期间，之后移除

**抛光是关键区别**：最后 20%——缓动曲线、时间偏移、次要运动——将"酷"与"非凡"区分开来。

**浏览器验证是强制性的**：技术上有雄心的效果几乎从来不会第一次尝试就工作。预览、验证和迭代。

---

## 绝不

- 忽略 `prefers-reduced-motion`——提供美丽的静态替代方案
- 发送在中档设备上引起卡顿的效果
- 使用没有功能性后备的前沿 API
- 没有明确用户选择加入就添加声音
- 使用技术野心来掩盖弱的设计基础
- 层叠多个竞争的"非凡"时刻——选择一个 hero 效果

## 验证结果

**Wow 测试**：有一个真正的愉悦或惊喜时刻吗？
**移除测试**：如果移除效果，设计会更糟吗？
**设备测试**：它在中档 Android 手机上运行流畅吗？
**无障碍测试**：它适用于 `prefers-reduced-motion: reduce`？
**上下文测试**：这个野心水平匹配项目的个性和目标吗？
