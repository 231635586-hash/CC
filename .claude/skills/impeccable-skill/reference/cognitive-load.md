# Cognitive Load Assessment（认知负荷评估）

认知负荷是使用界面所需的总心理努力。负担过重的用户会犯错、感到沮丧并离开。

---

## 三种认知负荷类型

### 内在负荷：任务本身
用户试图做的复杂性固有。你不能消除这个，但你可以结构化它。

**通过以下方式管理**：将复杂任务分解为离散步骤；提供脚手架（模板、默认值、示例）；渐进披露；将相关决策分组在一起。

### 外部负荷：糟糕的设计
由糟糕的设计选择引起的心理努力。**无情地消除这个。** 这是纯粹的浪费。

**常见来源**：混乱的导航；不清楚的标签迫使用户猜测；视觉混乱；不一致的模式；用户意图和结果之间的不必要步骤。

### 相关负荷：学习努力
用于建立理解的心理努力。这是*好的*认知负荷——它导致掌握。

**通过以下方式支持**：逐渐揭示复杂性的渐进披露；奖励学习的一致模式；确认正确理解的反馈。

---

## 认知负荷检查表

- [ ] **单一焦点**：用户可以在不分心的情况下完成主要任务吗？
- [ ] **分块**：信息分组可消化（每组 ≤4 项）？
- [ ] **分组**：相关项目视觉分组（邻近、边框、共享背景）？
- [ ] **视觉层级**：最重要的内容是否立即清楚？
- [ ] **一次一件事**：用户可以在进入下一步之前专注于单一决策？
- [ ] **最少选择**：决策简化（任何决策点 ≤4 个可见选项）？
- [ ] **工作记忆**：用户需要记住先前屏幕的信息吗？
- [ ] **渐进披露**：复杂性仅在需要时揭示？

**评分**：0–1 个失败 = 低（好）。2–3 = 中等（尽快解决）。4+ = 关键修复需要。

---

## 工作记忆规则

人类一次可以在工作记忆中保存 ≤4 个项目（Cowan, 2001 修订的 Miller's Law）。

- **≤4 个项目**：在工作记忆限制内
- **5–7 个项目**：触及边界；考虑分组或渐进披露
- **8+ 个项目**：超载；用户会跳过、误点击或放弃

**实际应用**：
- 导航菜单：≤5 个顶级项
- 表单部分：在视觉休息之前每组 ≤4 个字段
- 操作按钮：1 个主要，1–2 个次要，其余分组在菜单中
- 仪表盘小部件：≤4 个关键指标在不滚动的情况下可见
- 定价层级：≤3 个选项（更多导致分析瘫痪）

## 常见违规

**选项墙**：10+ 个选择没有层级 → 分组，突出推荐，使用渐进披露。

**记忆桥**：用户必须从第 1 步记住信息才能完成第 3 步 → 在需要的地方保持上下文可见。

**隐藏导航**：用户必须 mental map 某物在哪里 → 显示当前位置（面包屑、活动状态、进度指示器）。

**行话屏障**：技术语言强制翻译努力 → 使用简单语言；在行内定义领域术语。

**视觉噪音 floor**：每个元素具有相同的权重 → 建立清晰的层级：一个主要，2–3 个次要，其余柔和。

**不一致模式**：相似操作在不同地方以不同方式工作 → 标准化。相同类型的操作 = 相同类型的 UI。

**多任务需求**：界面需要处理多个同时输入 → 排序步骤。

**上下文切换**：用户必须在屏幕之间跳转以收集单个决策的信息 → 将每个决策所需的信息共置。
