# Impeccable Live Mode（实时模式）

通过浏览器中的 AI 辅助工作流进行交互式设计变体生成。

---

## 核心工作流程

1. **初始化**：通过 `live.mjs` 启动辅助服务器
2. **导航**：打开 app URL（绝不要直接使用辅助端口）
3. **轮询循环**：重复执行 `live-poll.mjs` 以处理事件
4. **事件处理**：响应生成、引导、接受/丢弃请求
5. **清理**：停止服务器并移除注入标签

---

## 关键约束

- 不能跳过步骤——按确切顺序执行合同
- 长轮询：绝不要使用短超时（默认 600,000ms）
- 每次生成一次编辑：在单次文件修改中写入所有三个变体
- 在规划变体之前保留现有视觉 identity
- 每个变体暴露 2–3 个可调轴

---

## 设计生成过程

处理 `generate` 事件时：
1. 如果存在截图则读取注释
2. 使用确定性辅助函数包装元素
3. 从 CSS 变量、计算样式和兄弟元素中提取 identity
4. 沿不同主轴（层级、布局、字体排版、颜色、密度、结构）计划三个变体
5. 在一次编辑中写入带作用域 CSS 的完整 HTML 替换
6. 在变体包装器上声明参数为 JSON 元数据

---

## 按视觉重量的参数策略

- 叶元素：0 个参数
- 小组合：0–1 个参数
- 中等组合：目标 2 个参数
- 大组合：2–3 个参数（最多 4 个）

---

## 配置

首次设置需要 `.impeccable/live/config.json`，包含文件目标、CSP 补丁和语法检测。
