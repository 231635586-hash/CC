# Extract Flow：设计系统整合

识别并整合可复用的 UI 模式到设计系统中。

---

## 关键流程步骤

1. **发现与规划**：定位现有设计系统基础设施。如果没有设计系统存在，停止并与用户澄清，然后再创建一个。

2. **模式识别**：专注于以一致意图出现 3+ 次的 UI 元素——重复组件、硬编码值、组合模式。

3. **提取规划**：记录要提取哪些组件、要创建的设计 tokens、必要的命名约定。

4. **实现**：用适当的 API、变体、无障碍功能、文档构建增强的可复用版本。

5. **迁移与测试**：系统地用共享版本替换现有实现，同时保持视觉和功能对等。

6. **文档**：用新组件、tokens、示例和使用指南更新设计系统记录。

---

## 关键原则

"过早的抽象比重复更糟糕。"不要创建过度通用的组件或不适当概括化就提取上下文特定实现。

优先增量增长——提取明显可复用的模式，而不是推测的未来用例。
