# Color & Contrast（颜色与对比度）

---

## 色彩空间：使用 OKLCH

**停止使用 HSL。** 使用 OKLCH 代替。它是感知均匀的—— lightness 的相等步骤*看起来*相等，不像 HSL 中黄色 50% lightness 看起来明亮而蓝色 50% lightness 看起来暗。

`oklch(lightness chroma hue)` — lightness 0–100%，chroma 大约 0–0.4，hue 0–360。

要构建具有更浅/更深变体的主色：保持 chroma+hue 常数，改变 lightness——但**在接近白色或黑色时减少 chroma**（极端 lightness 的高 chroma 看起来刺眼）。

Hue 是品牌决定。不要本能地选择蓝色（hue 250）或暖橙色（hue 60）——那些是 AI 设计默认值，不是任何特定品牌的正确答案。

## 构建功能性调色板

### 着色中性色

**纯灰色是死的。** 向所有中性色添加微小的 chroma 值（0.005–0.015），向品牌颜色着色。Chroma 小到不足以在意识上读作"着色"，但创造下意识凝聚力。Hue 应该来自这个项目的品牌——不是来自"暖 = 友好，冷 = 科技"公式。

### 调色板结构

| 角色 | 目的 | 示例 |
|------|---------|---------|
| **Primary** | 品牌、CTA、关键操作 | 1 种颜色，3–5 个色度 |
| **Neutral** | 文本、背景、边框 | 9–11 色度比例 |
| **Semantic** | 成功、错误、警告、信息 | 4 种颜色，每种 2–3 个色度 |
| **Surface** | 卡片、模态框、覆盖层 | 2–3 个海拔级别 |

除非需要，否则跳过次要/第三级。大多数应用程序一种 accent 颜色就很好。

### 60-30-10 规则（正确应用）

这是关于**视觉重量**，不是像素计数：
- **60%**：中性背景、留白、基础表面
- **30%**：文本、边框、非活动状态
- **10%**：Accent — CTA、高亮、焦点状态

过度使用会削弱 accent 力量。

## 对比度与无障碍

### WCAG 要求

| 内容类型 | AA 最低 | AAA 目标 |
|--------------|------------|-----------|
| 正文文本 | 4.5:1 | 7:1 |
| 大文本（18px+ 或 14px 粗体） | 3:1 | 4.5:1 |
| UI 组件、图标 | 3:1 | 4.5:1 |
| 非必要装饰 | 无 | 无 |

占位符文本仍然需要 4.5:1。那种浅灰色占位符到处都有？通常失败。

### 危险组合

- 浅灰色文本在白色上（#1 无障碍失败）
- **任何彩色背景上的灰色文本**——使用背景颜色的深色阴影代替
- 红色文本在绿色上（或反之）——8% 的男性无法区分这些
- 蓝色文本在红色上（视觉振动）
- 黄色文本在白色上（几乎总是失败）
- 图像上的浅色细文本（不可预测的对比度）

### 绝不使用纯灰色或纯黑色

纯灰色和 `#000` 在自然界中不存在。0.005–0.01 的 chroma 足以感觉自然。深色模式背景也一样——使用深灰色（oklch 12–18%），绝不是纯黑色。

### 测试

不要相信你的眼睛——使用工具：WebAIM Contrast Checker、浏览器 DevTools Rendering → 模拟视力缺陷、Polypane 用于实时测试。

## 主题化：浅色与深色模式

深色模式不是反转的浅色模式。你不能只是交换颜色。

| 浅色模式 | 深色模式 |
|------------|-----------|
| 用于深度的阴影 | 用于深度的较浅表面（无阴影） |
| 浅色上的深色文本 | 深色上的浅色文本（略微减少字重） |
| 鲜艳 accent | 略微取消饱和 accent |
| 白色背景 | 深灰色（oklch 12–18%），绝不是纯黑色 |

在深色模式中，深度来自表面 lightness。构建 3 步表面比例，其中较高海拔较浅（例如 15% / 20% / 25%）。略微减少正文字重（350 而不是 400）——深色上的浅文本阅读起来更重。

### Token 层级

两层：原语 tokens（`--blue-500`）和语义 tokens（`--color-primary: var(--blue-500)`）。对于深色模式，仅重新定义语义层。

---

## Alpha 是设计气味

严重的透明度（rgba、hsla）通常意味着不完整的调色板。Alpha 产生不可预测的对比度和不一致。为每个上下文定义明确的覆盖颜色。例外：需要透明的焦点环和交互状态。

**避免**：仅依赖颜色传达信息。大面积使用纯黑色。跳过色盲测试。
