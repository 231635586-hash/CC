# Technical Quality Audit（技术质量审计）

> 运行系统化技术质量检查并生成全面报告。不要修复问题；为其他命令记录问题以待处理。

这是代码级别的审计，不是设计评审。检查实现中可测量和可验证的内容。

---

## 诊断扫描

每个维度评分 0–4。

### 1. 可访问性（A11y）

检查：
- **对比度**：文本对比度 < 4.5:1（或 AAA 7:1）
- **缺少 ARIA**：交互元素没有适当的角色、标签或状态
- **键盘导航**：缺少焦点指示器、不符合逻辑的 Tab 顺序、键盘陷阱
- **语义 HTML**：标题层级不当、缺少地标、用 div 代替按钮
- **Alt 文本**：图片描述缺失或质量差
- **表单**：输入框没有标签、错误消息差、缺少必填指示器

**分数**：0 = 不可访问（WCAG A 失败）· 1 = 重大缺口 · 2 = 部分 · 3 = 良好（大部分满足 WCAG AA）· 4 = 卓越（完全满足 WCAG AA，接近 AAA）

### 2. 性能（Performance）

检查：
- **布局抖动**：在循环中读取/写入布局属性
- **昂贵动画**：布局属性动画、无界 blur/filter/shadow、帧率下降
- **缺少优化**：图片没有懒加载、未优化资源、缺少 `will-change`
- **包大小**：不必要的导入、未使用的依赖
- **渲染性能**：不必要的重新渲染、缺少记忆化

**分数**：0 = 严重（布局抖动，一切未优化）· 1 = 重大 · 2 = 部分 · 3 = 良好 · 4 = 卓越（快速、精简、优化良好）

### 3. 主题化（Theming）

检查：
- **硬编码颜色**：颜色没有使用设计 tokens
- **破碎的深色模式**：缺少深色模式变体、深色主题对比度差
- **不一致的 tokens**：错误的 tokens、混合 token 类型
- **主题切换**：主题更改时不更新的值

**分数**：0 = 无主题化 · 1 = 最小 tokens · 2 = 部分 · 3 = 良好 · 4 = 卓越（完整 token 系统，深色模式完美工作）

### 4. 响应式设计（Responsive Design）

检查：
- **固定宽度**：在移动端破坏的硬编码宽度
- **触摸目标**：交互元素 < 44×44px
- **水平滚动**：窄视口内容溢出
- **文本缩放**：文本大小增加时破坏的布局

**分数**：0 = 不响应 · 1 = 重大问题 · 2 = 部分 · 3 = 良好 · 4 = 卓越（所有断点完美适配）

### 5. 代码质量（Code Quality）

检查：
- **组件结构**：组件是否良好分离、可复用
- **状态管理**：状态是否适当管理，有没有 prop drilling
- **错误处理**：错误状态是否有清晰的处理
- **代码可读性**：代码是否有适当注释和文档

**分数**：0 = 混乱 · 1 = 重大问题 · 2 = 部分 · 3 = 良好 · 4 = 卓越
