# Layout：修复间距、节奏和视觉层级

> "找到布局的实际问题（单调间距、弱层级、相同的卡片网格、居中堆叠默认），并修复结构，而不是表面。"

布局问题通常是界面感觉'不对劲'的根本原因，即使颜色和字体都很好。空间是一种设计材料——有意识地使用它。

---

## 注册类型

**Brand（品牌型）**：不对称构图，用 `clamp()` 实现流体间距，为强调而刻意打破网格。

**Product（产品型）**：可预测的网格、一致的密度、熟悉的导航模式。"一致性本身是一种可供性。"

---

## 评估当前布局

### 1. 间距
- 间距是一致的还是任意的？
- 所有间距都相同吗？（单调）
- 相关元素是否紧密分组，组之间有充足的空间？

### 2. 视觉层级
- 眯眼测试：眯眼时能识别层级吗？
- 留白是否引导眼睛看向重要的东西？
- 层级是通过多个维度还是仅通过大小实现的？

### 3. 网格与结构
- 有清晰的底层结构吗？
- 是否到处使用相同的卡片网格？
- 一切都居中吗？（默认居中堆叠不是布局）

### 4. 节奏与变化
- 布局有视觉节奏吗？
- 每个节的结构都相同吗？
- 有刻意的惊喜或强调时刻吗？

### 5. 密度
- 对于内容类型来说太拥挤还是太稀疏？
- 密度与显示的内容匹配吗？

## 系统性地改善布局

### 建立间距系统

使用 4pt 基准刻度：`4, 8, 12, 16, 24, 32, 48, 64, 96px`

用语义命名 tokens（`--space-component-gap`，不是 `--space-16`）。用 `gap` 处理同级间距。在断点之间用 `clamp()` 实现流体间距。

### 创造视觉节奏

- 相关元素紧密分组（8–12px）
- 节之间慷慨分隔（48–96px）
- 节内间距变化（不统一）
- 不对称构图（品牌）或一致密度（产品）

### 选择正确的布局工具

- **Flexbox**：一维布局（行或列）
- **Grid**：二维布局，需要同时控制行和列
- 当 Flexbox 足够时不要默认使用 Grid
- 自适应网格：`repeat(auto-fit, minmax(280px, 1fr))`
- 复杂布局的命名网格区域

### 打破卡片网格单调

不要无限制地重复相同的卡片大小和结构。混合：
- 不同的卡片大小（hero 卡片 + 较小卡片）
- Bento 风格不对称网格
- 列表/行布局与网格节交替
- 卡片组之间的全出血节

### 加强视觉层级

使用建立清晰层级所需的最少维度。仅仅空间通常就足够了——元素周围慷慨的留白吸引眼球，而不需要大小变化。结合维度时：大小 + 字重 + 颜色 + 空间。

注意阅读流（F 模式、Z 模式）并创建清晰的内容分组。

### 管理深度与海拔

创建语义化 z-index 比例（不是任意数字）。构建与海拔级别关联的 consistent 阴影比例。使用海拔来加强层级，而不是装饰。

### 光学调整

如果一个图标或元素尽管数学上间距正确看起来还是不对，相信你的眼睛——微调它。光学对齐 ≠ 数学对齐。

---

## 绝不要

- 使用任意间距值
- 使所有间距处处相等
- 把所有东西都用卡片包装
- 在卡片内嵌套卡片
- 到处使用相同的卡片网格
- 默认让一切居中
- 当 Flexbox 足够时默认使用 CSS Grid 而不是 Flexbox
- 使用任意的 z-index 值（`z-index: 9999`）

---

## 验证

- 眯眼测试：2 秒内可识别层级
- 节奏：组内紧密，节之间慷慨
- 结构：清晰的底层网格或系统
- 呼吸空间：没有感觉拥挤或漂浮的东西
- 一致性：间距遵循定义的刻度
- 响应式：优雅组合，不只是收缩
