# Harden：面向边界情况的生产就绪

> "只对完美数据工作的设计不是生产就绪的。使界面硬化，以抵御真实用户会扔给它的输入、错误、语言和网络条件。"

> "硬化是关于预期意外。真实用户会做出你从未想象的事情。"

---

## 硬化维度

### 文本溢出与换行

- 通过截断、行截断和换行处理溢出——不是隐藏的切断
- 在 flex 项目上使用 `min-width: 0` 以防止溢出
- 用非常长的文本、非常短的文本、全大写、emoji、数字测试
- 绝不要对文本容器应用固定宽度
- 为单行 `text-overflow: ellipsis`、多行 `line-clamp` 做计划

### 国际化（i18n）

- 为翻译预留 30–40% 额外空间（德语、芬兰语、荷兰语扩展显著）
- 使用逻辑 CSS 属性（`margin-inline-start` 而不是 `margin-left`）以支持 RTL
- 支持 UTF-8 编码和 CJK 字符集
- 使用 Intl API 进行日期、时间、数字和货币格式化
- 用 RTL 语言（阿拉伯语、希伯来语）测试——布局应该正确镜像
- 绝不要硬编码日期格式、货币符号或数字分隔符

### 错误处理

清晰、具体的错误消息和恢复选项。按 HTTP 状态区分：
- **400**："你的输入有问题" + 哪个字段
- **401**："请登录" + 登录链接
- **403**："你没有访问权限" + 如何获取
- **404**："未找到" + 导航选项
- **429**："请求过多，请等待 X 秒"
- **500**："我们这边出了问题" + 重试选项

绝不要使用通用的"发生错误"。绝不要责怪用户。

### 边界情况与边界条件

明确为以下设计：
- **空状态**：每个列表、表格和 feed 都需要空状态
- **加载**：每个异步操作都需要加载状态
- **大数据集**：1000+ 项——虚拟化、分页或懒加载
- **并发操作**：多次同时编辑或表单提交
- **权限限制**：元素存在但用户无法访问

### 输入验证与清理

- 为 UX 进行客户端验证；**始终**为安全进行服务器端验证——"绝不要只信任客户端"
- 用 HTML 属性设置清晰约束（`maxlength`、`pattern`、`min`、`max`）
- 在正确时刻显示验证反馈（不是在第一次按键时）
- 清理用户生成内容的显示（XSS 防护）

### 可访问性弹性

- 键盘导航必须适用于所有交互元素
- 屏幕阅读器支持：ARIA 标签、角色、动态内容的 live regions
- 处处尊重 `prefers-reduced-motion`
- 高对比度模式兼容
- 文本可缩放到 200% 而不破坏布局
