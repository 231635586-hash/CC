# 🚨 底座冻结期公告 — hx-archive-system

> **生效时间**: 2026-07-08 至 2026-08-08（30 天）
> **冻结对象**: hx-archive-system（组织管理）底座代码
> **公告目的**: 给上层业务模块一个稳定依赖期

---

## ❄️ 冻结范围

### 不接受（拒绝）
- ❌ 新增 feature
- ❌ 公共组件 API 变更
- ❌ 既有架构重构
- ❌ 跨底座的依赖升级

### 接受（正常处理）
- ✅ Bug 修复（含 P0/P1/P2 各级）
- ✅ 安全补丁
- ✅ 阻塞业务的紧急修复（需用户明确授权）
- ✅ 文档/注释更新

---

| 业务模块 | 冻结期策略 |
|----------|-----------|
| hx-archive-system（组织管理）业务代码 | 🟢 正常迭代，**但不得调用底座新 API** |
| hx-logistics-system（物流管理） | 🟢 正常迭代，**但不得调用底座新 API** |
| hx-order-system | 🟡 观察态，本期不受影响 |
| mes-workorder | 🟢 独立项目，不受影响 |

## 🔄 解除条件（2026-08-08 评估）

评估维度：
1. 过去 30 天底座 bug 数量
2. 业务模块调用底座 API 的频率
3. 是否有阻塞性技术债积累
4. 团队对底座稳定性的主观评价

**评估结果 3 种可能**：
- ✅ 通过 → 恢复迭代，进入下一阶段
- ⚠️ 部分通过 → 继续冻结 15-30 天
- ❌ 未通过 → 继续冻结 30 天 + 紧急重构

## 🆘 紧急情况处理

如果冻结期内出现必须变更底座的紧急情况：

2. 提交"紧急解冻申请"段落在本公告下方
3. 用户口头/书面授权后方可变更
4. 变更后追加 commit 说明"紧急解冻 - [原因]"
