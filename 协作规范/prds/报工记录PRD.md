# 快速PRD：报工记录迭代 v1.0

> **生成日期**: 2026-07-08
> **PRD 模式**: 快速PRD（11 项主要功能，含 6 项 P0 + 3 项 P1 + 2 项 P1）
> **产品类别**: B 端（企业内部系统 - 扫码报工）
> **关键决策**: 岗位 = 工作点+岗位类型组合 / v1 不考虑审批异常 / 独立 PRD 不绑定本地项目

## 1. 概述

针对扫码报工系统的"报工记录"模块进行功能迭代，核心 5 大方向：
- 🟢 **流程升级**: 新增下班班长审批流
- 🟢 **数据治理**: 强化唯一性约束 + 字段优化
- 🟢 **体验优化**: 审批人记忆 + 移动端日期快捷选择
- 🟢 **审计能力**: 引入变更记录模块
- 🟢 **配置升级**: 生产组织导入字段调整 + 二维码批量导出

**目标用户**:
- 一线工人（扫码报工）
- 班长（审批）
- 车间管理员（报工点配置）

## 3. 验收标准 (AC)

### AC-01 下班审批（P0）
- [ ] 工人扫码下班后，记录状态变为「待班长审批」
- [ ] 班长的待审批列表显示该记录
- [ ] 班长点击「通过」→ 状态「已审批」，工作时长自动计算
- [ ] 班长点击「驳回」→ 状态「已驳回」，工人需重新扫码

### AC-02 唯一性约束（P0）
- [ ] 同一用户同一天同一（工作点+岗位类型）组合扫码时，系统拦截
- [ ] 错误提示：「您今日在 X 工作点的 Y 岗位已有报工记录」
- [ ] 引导用户查看已有记录，不创建新记录

### AC-03 审批人记忆（P0）
- [ ] 首次选择审批人 A，下次进入页面默认选中 A
- [ ] 切换审批人为 B，后续默认 B
- [ ] 记忆范围：当前用户 + 当前设备（LocalStorage）

### AC-04 字段优化 - 删除（P0）
- [ ] 列表页不再显示「报工记录编码」列
- [ ] 详情页同步移除该字段

### AC-05 字段优化 - 新增（P0）
- [ ] 列表页新增「工作日期/上班时间/下班时间/工作时长」4 列
- [ ] 工作时长 = 下班时间 - 上班时间（小时，保留 1 位小数）
- [ ] 下班时间未填时工作时长显示「-」

### AC-06 编辑功能（P0）
- [ ] 详情页新增「编辑」按钮
- [ ] 点击后弹出编辑弹窗，可改「报工点/班次/工作日期」3 个字段
- [ ] 其他字段（审批状态/审批人/上下班时间）不可编辑
- [ ] 编辑保存后，弹窗询问「是否需要重新提交审批」

### AC-07 变更记录模块（P0）
- [ ] 详情抽屉底部新增「变更记录」模块
- [ ] 记录格式：`[时间] [操作人] 修改 [字段名]：[旧值] → [新值]`
- [ ] 仅「编辑」操作产生记录，其他操作不记录

### AC-08 生产组织配置（P1）
- [ ] 导入模板的「手机号」列改为「工号」
- [ ] 字段提示文案同步更新
- [ ] v1 历史数据无需迁移

### AC-09 移动端日期选择（P1）
- [ ] 移动端「工作日期」字段从日期选择器改为快捷 3 选项
- [ ] 选项：`[今日 / 昨日 / 明日]`
- [ ] 选择后显示对应日期文本（如「今日（2026-07-08）」）

### AC-10 二维码导出（P1）
- [ ] 报工点配置页「导出二维码」点击后，下载 .zip 压缩包
- [ ] 压缩包内含该工作点下所有报工点的二维码图片
- [ ] 文件命名规则：`[报工点名称].png`

## 8. 边界场景

| 场景 | 页面表现 | 用户提示 | 处理逻辑 |
|------|----------|----------|----------|
| 重复扫码（违反唯一约束） | 弹窗 | "您今日在 X 工作点的 Y 岗位已有报工记录" | 引导查看已有记录，**不创建新记录** |
| 审批人记忆失效（用户换设备） | 审批人下拉默认空 | 提示 "请选择审批人" | 用户正常选择即可 |
| 班次未选择提交 | 表单校验 | "请选择班次" | 阻塞提交 |
| 跨天扫码（00:00-06:00 仍按前一天） | 工作日期选「昨日」 | 提示 "当前为凌晨，是否按昨日记录？" | 用户确认 |
| 编辑后无需重新审批 | 弹窗 | "是否需要重新提交审批？" | 用户选「否」则保留已审批状态 |
| 二维码压缩包下载失败 | Toast | "导出失败，请重试" | 重试按钮 |
| 工作时长异常（>16 小时） | 列表显示红色 | 无 | 数据层面校验，非阻塞 |
| 用户无审批人可选（无配置班长） | 下拉空 + 提示 | "请联系管理员配置班长" | 阻塞提交，联系管理员 |
| 变更记录超过 100 条 | 时间线滚动加载 | 无 | 性能优化：分页 20 条/页 |
| 工人下班忘记打卡 | 列表显示「未完成」标记 | 无 | v1 不强制处理，可手动补录 |
