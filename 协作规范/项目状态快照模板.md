# 项目状态快照模板

> **使用方式**：每次开新会话，开头粘贴一次本模板并填好。
> **目的**：让 AI 协作方快速进入状态，避免每次从零建立 context。

---

## 📋 快照模板

```markdown
## 项目: [项目名]
## 当前版本: [vX.Y.Z]
## 主线任务: [本周最关键 1 件事，用一句话说]
## 进行中: 
  - [任务 A 状态]
  - [任务 B 状态]
  - [任务 C 状态]
## 待解决: 
  - [问题 1]
  - [问题 2]
## 阻塞项: 
  - [阻塞 + 谁来解决]
## 上次更新: [YYYY-MM-DD]
## 当前会话目标: [本次会话要交付什么]
```

---

## 📝 填写示例

```markdown
## 项目: hx-archive-system（组织管理）
## 当前版本: v1.0.6
## 主线任务: 派车调度模块稳定性优化
## 进行中: 
  - 派车 Modal 司机/车辆下拉接入 ✅ 已完成
  - 调车单作废/删除状态机 ✅ 已完成
  - 司机档案模块 ✅ 已完成
## 待解决: 
  - 派车调度实时刷新偶发不触发
  - 车辆位置 4 列信息需 UI 微调
## 阻塞项: 
  - 无
## 上次更新: 2026-07-08
## 当前会话目标: 排查派车调度实时刷新问题
```

---

## 🎯 填写规则

1. **5 分钟填完**：快照应该 < 5 分钟搞定，复杂了就过度细节
2. **3-5 条进行中**：超过 5 条说明没主线，需要砍
3. **阻塞项写"谁"**：写"谁来解决"而不是只写"等 XX"，逼自己定义责任
4. **不写历史**：快照只反映"现在"，历史去 git log 看
