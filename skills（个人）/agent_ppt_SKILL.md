# Apple Style PPT Skill

设计理念遵循"少即是多"的极简主义。每页突出一个主题，整份演示文稿布局、字体、配色和谐统一。

## 设计规范

### 配色方案

| 角色 | 色值 | 用途 |
|------|------|------|
| 主背景 | #FFFFFF 纯白 | 页面底色 |
| 主文字 | #1D1D1F 深灰黑 | 标题、正文 |
| 次要文字 | #86868B 中灰 | 副标题、标注、注释 |
| 点缀色 | #0071E3 Apple蓝 | 关键数据、强调、链接、按钮 |
| 点缀亮色 | #47A3FF | 深色背景上的强调文字 |
| 浅蓝辉光 | rgba(0,113,227,0.03-0.07) | 背景径向渐变装饰 |
| 灰色卡片底 | #F5F5F7 | 浅灰卡片 |
| 深色背景 | #1D1D1F | 章节页/强调内容页深色底 |
| 深色副背景 | #2D2D2F | 深色卡片次级底 |

### 字体

| 层级 | 字体 | 字号 | 字重 | 备注 |
|------|------|------|------|------|
| 大标题 | PingFang SC / SF Pro | 48-64px | Semibold 600 | 封面/章节页 |
| 页面标题 | PingFang SC / SF Pro | 34-40px | Bold 700 | 内容页标题 |
| 副标题/摘要 | PingFang SC / SF Pro | 18-20px | Regular 400 | 一句话说明 |
| 正文 | PingFang SC / SF Pro | 13-16px | Regular 400 | 主要内容 |
| 标签/指标 | PingFang SC / SF Pro | 10-12px | Medium 500 | badge/chip 标签 |
| 标注/脚注 | PingFang SC / SF Pro | 11-12px | Regular 400 | 数据来源、注释 |

> 中文优先用 **苹方 (PingFang SC)**，英文/数字用 **San Francisco (SF Pro)**。
> 备选：Noto Sans SC / Helvetica Neue / Inter。

### 布局规范

- **留白优先**：内容区域占页面 ≤60%，四周留白 ≥15%
- **对齐原则**：统一左对齐或居中对齐，不混用
- **间距标准**：元素间距 8px 倍数（8/16/24/32/40/64/80px）
- **每页1个核心信息**：一句话能说清这页在讲什么
- **深浅交替节奏**：每 2-3 个内容页后插入一个深色冲击页，打破视觉疲劳

## 动画交互方案

所有演示文稿支持 6 种可选的动画交互方案。用户在设计阶段 **选择组合**，每 2-3 页切换一种以避免审美疲劳。

### 方案 A · 粘性视差（Sticky Parallax）
| 属性 | 说明 |
|------|------|
| 适合页 | 封面、章节过渡、价值总结 |
| 效果 | 核心文字固定在视口中心，背景卡片浮动淡入 |
| 实现 | CSS position:sticky \| Keynote 神奇移动 \| PPT 平滑过渡 |
| 体验 | ⭐⭐⭐⭐⭐（沉浸感最强） |

### 方案 B · 逐帧滚入（Scroll Reveal）
| 属性 | 说明 |
|------|------|
| 适合页 | 痛点列表、能力展示、路线图 |
| 效果 | 每个条目按序逐一淡入 + 上移，延迟阶梯递增 |
| 实现 | IntersectionObserver + transition-delay \| 逐项构建 |
| 体验 | ⭐⭐⭐⭐（叙事感最强） |

### 方案 C · 层级揭幕（Layered Reveal）
| 属性 | 说明 |
|------|------|
| 适合页 | 关键指标、AI 引擎、数据展示 |
| 效果 | 辉光→大数字(scale)→标题(fade+up)→描述→标签，逐层揭晓 |
| 实现 | CSS scale + opacity + transition-delay 链式 |
| 体验 | ⭐⭐⭐⭐⭐（节奏感最强） |

### 方案 D · 分屏滑入（Split-Screen Slide In）
| 属性 | 说明 |
|------|------|
| 适合页 | 对比页（新旧对比、竞品对比） |
| 效果 | 左右两侧从视口外滑入，VS 徽标延迟放大出现 |
| 实现 | translateX 双向滑入 + 延迟缩放 |
| 体验 | ⭐⭐⭐⭐⭐（对比冲击力最强） |

### 方案 E · 数据递增（Counting Animation）
| 属性 | 说明 |
|------|------|
| 适合页 | 价值总结、成果展示、KPI 汇总 |
| 效果 | 数字从 0 滚动至目标值，进度条同步填充 |
| 实现 | requestAnimationFrame + easeOutCubic |
| 体验 | ⭐⭐⭐⭐（参与感最强） |

### 方案 F · 沉浸序列（Cinematic Sequence）
| 属性 | 说明 |
|------|------|
| 适合页 | 产品故事、路线图、开场序列 |
| 效果 | 全屏背景渐变切换 + 文字缩放 + 时序标签递进 |
| 实现 | 场景状态机 + CSS background + 文字属性过渡 |
| 体验 | ⭐⭐⭐⭐⭐（叙事节奏最强） |

### 推荐组合
```
封面 →    方案 A（粘性视差，沉浸开场）
痛点 →    方案 B（逐帧滚入，叙事推进）
能力页 →  方案 C（层级揭幕，冲击力）
对比页 →  方案 D（分屏滑入，对比感）
价值总结 →方案 E（数据递增，参与感）
路线图 →  方案 F（沉浸序列，节奏感）
```

> **关键原则**：每 2-3 页切换动画类型。
> **禁止**：旋转飞入、棋盘切换、百叶窗、弹跳、闪烁等过时效果。

### 工作流程中的动画沟通

1. **框架确认**阶段：先输出内容大纲，无需考虑动画
2. **设计确认**阶段：选定各页的动画方案（从 A-F 中选择）
3. **制作输出**阶段：动画根据选型内嵌到最终输出中
4. **交互演示**阶段：HTML 预览会呈现完整的动画效果
5. **线上/投屏**阶段：根据输出平台自动降级（PPT 用平滑过渡，Keynote 用神奇移动，HTML 保留全部交互）

## 输出方式

支持以下输出途径（用户根据沟通结果选择）：

| 方式 | 适用阶段 | 特点 |
|------|----------|------|
| **HTML 预览** ⭐推荐 | 内容确认 + 动画选型 | 交互式、无需安装、可快速迭代 |

## 工作流程

1. **需求沟通** → 主题、受众、页数、输出方式
2. **内容框架** → 逐页大纲确认
3. **设计方案** → 页面模板组合 + 配色模式 + 动画选型

5. **终版交付** → 转为正式 Keynote/PPT 格式
6. **迭代优化** → 每次制作后复盘，持续更新此 skill

## ⚠️ 实际项目经验总结（常见陷阱）

以下教训来自一次完整的抽水蓄能项目管理平台 PPT 制作复盘：

### 陷阱 1：信息密度过高
**问题**：每页塞了 3-4 段长描述 + 小字指标，看似信息量大实则观众记不住任何东西。
**解法**：每页只放 **1 个核心信息**。主内容用大数字/大标题传递冲击力，剩余细节做成小标签 (badge/chip) 放在底部，或放到附录/口头讲解。
- ❌ 错误：卡片内写 3 句话 + 2 个数据点
- ✅ 正确：大数字 40px 作为视觉锚点 + 1 句描述 + 3 个标签

### 陷阱 2：卡片风格单一
**问题**：所有内容页都用灰底卡片排网格，看到第 3 页观众就疲劳了。
**解法**：**深浅交替节奏**。浅色渐变卡片 ↔ 深色背景卡片交替出现。每 2-3 个内容页后插一个深色章节页或深色数据页，打破节奏。
- 白底页 → 深色章节页 → 浅色指标页 → 深色卡片页 → 浅色对比页 → ...

### 陷阱 3：用表格展示技术细节
**问题**：带边框的表格（尤其是多列表格）更像产品文档，不是演示文稿。
**解法**：拆成 **视觉卡片 2×2 或 2×3 网格**。每个卡片独立设计，头部放模型名 badge（如 "LSTM+Attention"），底部放指标 chip（如 "Recall 94.7%"）。一眼扫完，审美在线。

### 陷阱 4：缺少视觉高潮
**问题**：整份 PPT 信息平铺，没有一页让观众"wow"。
**解法**：每隔 4-5 页安排一个 **视觉冲击页**：全深色背景 + 大数字 + 简单一句话。或左右分屏对比 + VS 徽标。或全屏驾驶舱模拟图。节奏要有高低起伏。

### 陷阱 5：对比页用双列表
**问题**：左列 5 条 vs 右列 5 条，单调且审美疲劳。
**解法**：**左右两半屏分色**。左半浅灰（传统模式，灰色调），右半深黑（智能平台，蓝色调）。中间浮 VS 圆形徽标。每条用 ✓/✕ 图标强化视觉区分。

### 陷阱 6：技术深度和可读性的平衡
**问题**：技术指标写太多观众看不懂，写太少显得"不够AI"。
**解法**：**分层呈现**。页面层级只放一句话概括 + 关键数字 + 3 个 tag。模型名和技术指标做成小 badge/chip，想看细节的人自然会被吸引。不要试图在幻灯片上写完所有的技术细节。

### 陷阱 7：渐变和装饰的运用
**经验**：适当的渐变背景（浅蓝→白）、径向辉光装饰圆、小圆点网格，能极大提升精致感且不喧宾夺主。纯白背景在投影时容易显得"廉价"。

### 陷阱 8：动画效果缺失或过度
**问题**：初版缺少动画显得"静态死板"，但过度使用（弹跳/旋转/闪烁）会显得廉价。
**解法**：参考上方动画方案表中选型，每 2-3 页切换一种。HTML 预览阶段即可嵌入完整的动画效果，与客户确认后再正式输出。

## HTML 交互导航设计

生成 HTML 演示文稿时，默认集成以下交互机制：

### 导航方式

| 方式 | 操作 | 说明 |
|------|------|------|
| **鼠标滚轮** | 滚动 | 默认滚动翻页 |
| **左右点击热区** | 点击左右两侧 | 悬停显示箭头图标，点击翻页 |
| **键盘快捷键** | `→` / `↓` / 空格 | 下一页 |
|  | `←` / `↑` | 上一页 |
|  | `Home` | 第一页 |
|  | `End` | 最后一页 |
| **右侧指示点** | 点击 | 直接跳转对应页 |

### 视觉设计原则

> **底部导航栏**：默认隐藏（`opacity: 0`，`pointer-events: none`），保留功能但不影响美观

- **点击热区**：透明底 + 渐变遮罩，鼠标悬停时 `opacity: 1` 显示箭头
- **右侧指示点**：固定在右侧中间，当前页高亮（放大 + 主题色）
- **进度条**：页面顶部 3px 主题色进度条，实时反映浏览进度
- **顶部导航**：固定透明背景 + 毛玻璃效果，含 Logo + 锚点链接 + PDF导出按钮
- **导出按钮**：位于导航栏右侧，图标 + 文字，悬停高亮

### 核心 JavaScript

```javascript
// 分页状态管理
let currentSlide = 0;
const slides = document.querySelectorAll('.slide');

// 跳转到指定页
function goToSlide(index) {
  currentSlide = index;
  slides[index].scrollIntoView({ behavior: 'smooth' });
}

// 键盘导航
document.addEventListener('keydown', (e) => {
  if (e.key === 'ArrowRight' || e.key === ' ') nextSlide();
  else if (e.key === 'ArrowLeft') prevSlide();
  else if (e.key === 'Home') goToSlide(0);
  else if (e.key === 'End') goToSlide(totalSlides - 1);
});

// IntersectionObserver 动画触发
const observer = new IntersectionObserver(entries => {
  entries.forEach(entry => {
    if (entry.isIntersecting) entry.target.classList.add('visible');
  });
}, { threshold: 0.2 });
```

### PDF 导出实现

利用浏览器原生 `window.print()` 配合 `@media print` 样式：

```css
@media print {
  nav, .scroll-indicator, .click-nav, .bottom-nav, .progress-bar {
    display: none !important;
  }

  .slide {
    min-height: 100vh;
    page-break-after: always;
  }

  .fade-up, .scale-in {
    opacity: 1 !important;
    transform: none !important;
  }
}
```

```javascript
document.getElementById('exportPdf').addEventListener('click', () => {
  window.print();
});
```

> **提示**：导出 PDF 时浏览器自动隐藏导航元素，每页幻灯片自动分页。
